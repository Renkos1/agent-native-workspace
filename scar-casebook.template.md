# Scar casebook — the survey's evidence layer

status: TEMPLATE (instantiate in your org's **private workspace** — never inside a
project repo; see the contamination rule)

One row per paid-for lesson, grouped by survey dimension. The survey asks transferable
questions; this casebook holds the receipts behind them. Feed it **the moment a lesson
gets paid for** — not at retrospectives, when the details have already blurred.

**Contamination rule**: consulted during founding conversations only, never copied into
a project repo — another project's facts sitting in a fresh repo get read as this repo's
facts. If a founding session cannot reach this workspace, copy only the relevant rows,
labeled "origin evidence, not this project's facts."

**Distillation rule**: when a scar generalizes, promote its lesson into the survey
question itself (edit your survey copy); the casebook row stays behind as the citation.
A lesson that lives only here helps one org; distilled into a question, it ships.
